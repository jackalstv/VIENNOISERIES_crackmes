Identification des symboles Commande utilisée : strings -d Trust_yourself Résultat : on identifie deux modes de fonctionnement via les labels detect_pipe_mode, compare_flag_direct et compare_serials. En mode pipe (stdin), le binaire compare directement l'input avec flag_enc à 0x40208a via un XOR 0x42. En mode interactif, il utilise un hash custom (username → serial).
Analyse du binaire avec objdump objdump -d Trust_yourself En mode pipe, la fonction compare_flag_direct lit l'input dans input_buf_2, applique input[i] XOR 0x42 et compare avec flag_enc[i] byte par byte sur 16 caractères. Le flag est donc flag_enc[i] XOR 0x42.
Extraction du flag avec gdb Breakpoint posé sur compare_flag_direct.loop à 0x40127c. rdi pointe sur flag_enc à 0x40208a. Dump des 16 bytes : 0x24 0x2e 0x23 0x25 0x39 0x1d 0x16 0x23 0x6f 0x33 0x35 0x23 0x1d 0x0a 0x7f 0x3f XOR 0x42 sur chaque byte donne la flag : flag{_Ta-qwa_H=}
